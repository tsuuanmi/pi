# Utility Functions

The package exports several utility functions for common patterns.

## StringEnum

Creates a string enum TypeBox schema compatible with providers that don't support `anyOf/const` patterns:

```typescript
import { StringEnum } from "@tsuuanmi/pi-ai";

const OperationSchema = StringEnum(["add", "subtract", "multiply", "divide"], {
  description: "The operation to perform",
  default: "add",
});
// Generates: { type: "string", enum: ["add", "subtract", "multiply", "divide"], description: "...", default: "add" }
```

Always prefer `StringEnum` over `Type.Enum` for tool parameters. Some providers (notably Anthropic) do not support `anyOf/const` patterns generated by `Type.Enum`.

## JSON Repair Parsing

`parseJsonWithRepair()` and `parseStreamingJson()` handle incomplete or malformed JSON from streaming tool call arguments:

```typescript
import { parseJsonWithRepair, parseStreamingJson } from "@tsuuanmi/pi-ai";

// Repair incomplete JSON from streaming
const partial = '{"path": "/src/index.ts", "content": "hello';
const result = parseJsonWithRepair(partial);
// result: { path: "/src/index.ts", content: "hello" }

// Streaming incremental parsing
for await (const event of stream) {
  if (event.type === "toolcall_delta") {
    const args = event.partial.content[event.contentIndex]?.arguments;
    // args is already best-effort parsed via parseStreamingJson
  }
}
```

These utilities are used internally for streaming tool call argument parsing and are exported for advanced use cases.

## Provider Environment Resolution

`getProviderEnvValue(name, env?)` resolves environment variables with scoped overrides and Bun sandbox fallbacks:

```typescript
import { getProviderEnvValue } from "@tsuuanmi/pi-ai";

// Resolution order:
// 1. Scoped env override (the ProviderEnv argument)
// 2. process.env
// 3. Bun sandbox fallback (/proc/self/environ)

const apiKey = getProviderEnvValue("ANTHROPIC_API_KEY", { ANTHROPIC_API_KEY: "per-request-key" });
```

This function is used internally for API key and proxy resolution. It handles the Bun compiled binary edge case where `process.env` may be empty inside Linux sandboxes. See [Provider Environment](utils/provider-env.md) for the credential lookup helpers (`findEnvKeys`, `getEnvApiKey`).

## HTTP Proxy Configuration

The library supports standard proxy environment variables for HTTP(S) requests:

| Variable | Description |
|----------|-------------|
| `HTTP_PROXY` | Proxy for HTTP requests |
| `HTTPS_PROXY` | Proxy for HTTPS requests |
| `NO_PROXY` | Hosts to exclude from proxying |

Proxy configuration is resolved per-request via `StreamOptions.env` or `process.env`. The `NO_PROXY` variable supports:

- `*` to bypass all proxying
- Hostname matching (exact and wildcard)
- Port-specific exclusions (`hostname:8080`)

## Session Resource Cleanup

```typescript
import { registerSessionResourceCleanup, cleanupSessionResources } from "@tsuuanmi/pi-ai";

// Register cleanup for provider connections
const unregister = registerSessionResourceCleanup((sessionId) => {
  webSocketPool.close(sessionId);
});

// On shutdown or session end
cleanupSessionResources("session-123");
```

Headers utilities:

```typescript
import { headersToRecord } from "@tsuuanmi/pi-ai";

// Convert a Headers instance to a flat Record<string, string>
const record = headersToRecord(response.headers);
```

See [HTTP Headers](utils/headers.md).

## Sanitize Unicode Surrogates

```typescript
import { sanitizeSurrogates } from "@tsuuanmi/pi-ai";

// Replace lone surrogates with U+FFFD replacement character
const safe = sanitizeSurrogates(text);
```

Used internally to sanitize content before sending to providers that reject unpaired surrogates.
